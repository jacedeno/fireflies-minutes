# Fireflies Meeting Minutes Routine

Automatically turn Fireflies transcripts into structured meeting minutes and fan out to GitHub, Trilium, and email.

- **Trigger:** Fireflies webhook → Cloudflare Worker (HMAC proxy) → Claude Code Remote Trigger → routine fires immediately.
- **Sinks:** GitHub repo (JSON + Markdown), Trilium note, HTML email via Resend.
- **Runtime:** Claude Code Routine running on Anthropic infrastructure. No local machine required.

## Architecture

```
Zoom meeting (Fireflies bot recording)
        |
        v
Fireflies finishes transcription
        | webhook POST (HMAC-signed)
        v
+----------------------------------+
|  Cloudflare Worker               |
|  webhook.geekendzone.net         |
|  - verify HMAC signature         |
|  - extract meetingId             |
|  - POST to Claude Code trigger   |
+----------------------------------+
        |
        v
+----------------------------------+
|  Claude Code Remote Trigger      |
|  (Anthropic-hosted URL)          |
|  - receive meetingId             |
|  - fetch transcript from FF      |
|  - reason -> JSON + MD + HTML    |
|  - PUT GitHub, POST Trilium,     |
|    POST Resend                   |
+----------------------------------+
        |
        +---> GitHub  (minutes/YYYY/MM/...)
        +---> Trilium (ETAPI note)
        +---> Resend  (HTML email)
```

## Environment variables

| Name | Stored in | Purpose |
|------|-----------|---------|
| `FIREFLIES_API_KEY` | Routine env | GraphQL queries to Fireflies |
| `FIREFLIES_WEBHOOK_SECRET` | Worker secret | HMAC verification of incoming webhooks |
| `ANTHROPIC_ROUTINE_URL` | Worker secret | Claude Code trigger fire endpoint |
| `ANTHROPIC_ROUTINE_TOKEN` | Worker secret | Bearer token for the trigger endpoint |
| `RESEND_API_KEY` | Routine env | Send HTML email |
| `RESEND_FROM_EMAIL` | Routine env | Verified sender on `support.cedeno.app` |
| `MEETING_RECIPIENT_EMAIL` | Routine env | `jacedeno@geekendzone.com` |
| `GITHUB_TOKEN` | Routine env | PAT with `repo` scope |
| `GITHUB_OWNER` | Routine env | GitHub username/org |
| `GITHUB_REPO` | Routine env | `fireflies-minutes` |
| `TRILIUM_BASE_URL` | Routine env | e.g. `https://notes.geekendzone.net` |
| `TRILIUM_ETAPI_TOKEN` | Routine env | ETAPI auth |
| `TRILIUM_PARENT_NOTE_ID` | Routine env | Parent note id for meeting notes |

## Setup

See [DEPLOYMENT.md](./DEPLOYMENT.md) for the full step-by-step. Short version:

1. In Claude Code web UI: create a Remote Trigger routine, note the URL and token.
2. `cd worker && wrangler deploy` with secrets: `FIREFLIES_WEBHOOK_SECRET`, `ANTHROPIC_ROUTINE_URL`, `ANTHROPIC_ROUTINE_TOKEN`.
3. Configure Fireflies webhook to point at the Worker URL.
4. Set up routine environment with all API keys.

## End-to-end test

1. Hold a short meeting on Zoom with the Fireflies bot.
2. Wait for Fireflies to finish transcribing (typically a few minutes).
3. Within ~2 minutes you should have:
   - A new commit under `minutes/YYYY/MM/`.
   - A new note under your Trilium parent note.
   - An HTML email at `MEETING_RECIPIENT_EMAIL`.

If nothing arrives, check the routine's run log in Claude Code, then `npx wrangler tail` on the Worker.

## Troubleshooting

- **Email lands in spam.** Confirm SPF / DKIM / DMARC for `support.cedeno.app` in the Resend dashboard.
- **Worker rejects webhooks (401).** `FIREFLIES_WEBHOOK_SECRET` set in Fireflies must equal the Worker secret.
- **Worker returns 502 (trigger failed).** Check `ANTHROPIC_ROUTINE_URL` and `ANTHROPIC_ROUTINE_TOKEN` are correct. Verify the routine exists in Claude Code web UI.
- **Routine logs `transcript not ready after retries`.** Fireflies webhook fired but transcript was not queryable after 3 attempts. Re-deliver the webhook from Fireflies dashboard (Settings > Developer > Webhooks > delivery log > re-send).
- **Trilium 401.** ETAPI token rotated or `TRILIUM_BASE_URL` wrong. Test with `curl -H "Authorization: $TOKEN" $URL/etapi/app-info`.
- **GitHub 422 on PUT.** File already exists with a different sha. The routine handles this; if it persists, check that `GITHUB_TOKEN` has `repo` scope.
- **Remote Trigger URL changed.** Delete and re-create the routine trigger in Claude Code. Update `ANTHROPIC_ROUTINE_URL` Worker secret. Redeploy the Worker.
