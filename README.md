# Fireflies Meeting Minutes Routine

Automatically turn Fireflies transcripts into structured meeting minutes and fan out to GitHub, Trilium, and email.

- **Trigger:** Fireflies webhook → Cloudflare Worker → Cloudflare KV queue → Claude routine (every 15 minutes).
- **Sinks:** GitHub repo (JSON + Markdown), Trilium note, HTML email via Resend.
- **Runtime:** Claude Code Routine running on Anthropic infrastructure. No local machine or self-hosted runner required.

## Architecture

```
Zoom meeting (Fireflies bot recording)
        │
        ▼
Fireflies finishes transcription
        │ webhook POST (HMAC-signed)
        ▼
┌──────────────────────────────────┐
│  Cloudflare Worker               │
│  webhook.geekendzone.com         │
│  - verify HMAC signature         │
│  - extract meetingId             │
│  - write to KV: pending:<id>     │
└──────────────────────────────────┘
        │
        ▼
Cloudflare KV namespace `fireflies-queue`
        │
        ▼ (every 15 min, cron */15 * * * *)
┌──────────────────────────────────┐
│  Claude Code Routine             │
│  prompt: CLAUDE.md               │
│  - GET  /queue                   │
│  - fetch transcript              │
│  - reason → JSON + MD + HTML     │
│  - PUT GitHub, POST Trilium,     │
│    POST Resend                   │
│  - DELETE /queue/<id>            │
└──────────────────────────────────┘
        │
        ├──► GitHub  (minutes/YYYY/MM/...)
        ├──► Trilium (ETAPI note)
        └──► Resend  (HTML email)
```

## Environment variables

| Name | Stored in | Purpose |
|------|-----------|---------|
| `FIREFLIES_API_KEY` | Routine secret | GraphQL queries to Fireflies |
| `FIREFLIES_WEBHOOK_SECRET` | Worker secret | HMAC verification of incoming webhooks |
| `WORKER_ADMIN_TOKEN` | Worker secret + Routine secret | Auth for routine→Worker queue API |
| `WORKER_BASE_URL` | Routine secret | e.g. `https://webhook.geekendzone.com` |
| `RESEND_API_KEY` | Routine secret | Send HTML email |
| `RESEND_FROM_EMAIL` | Routine secret | Verified sender on `geekendzone.com` |
| `MEETING_RECIPIENT_EMAIL` | Routine secret | `jacedeno@geekendzone.com` |
| `GITHUB_TOKEN` | Routine secret | PAT with `repo` scope |
| `GITHUB_OWNER` | Routine secret | GitHub username/org |
| `GITHUB_REPO` | Routine secret | `fireflies-minutes` |
| `TRILIUM_BASE_URL` | Routine secret | e.g. `https://trilium.geekendzone.com` |
| `TRILIUM_ETAPI_TOKEN` | Routine secret | ETAPI auth |
| `TRILIUM_PARENT_NOTE_ID` | Routine secret | Parent note id for meeting notes |

## Setup

See [DEPLOYMENT.md](./DEPLOYMENT.md) for the full step-by-step. Short version:

1. `cd worker && wrangler deploy` (creates and binds the KV namespace via `wrangler.toml`).
2. Configure Fireflies webhook → Worker URL with `FIREFLIES_WEBHOOK_SECRET`.
3. Verify the sending domain in Resend.
4. Create a Trilium ETAPI token and pick a parent note id.
5. In Claude Code: `/schedule` and paste `CLAUDE.md` as the prompt with cron `*/15 * * * *`. Add all routine secrets.

## End-to-end test

1. Hold a short meeting on Zoom with the Fireflies bot.
2. Wait for Fireflies to finish transcribing (typically a few minutes).
3. Within ~15 minutes you should have:
   - A new commit under `minutes/YYYY/MM/`.
   - A new note under your Trilium parent note.
   - An HTML email at `MEETING_RECIPIENT_EMAIL`.

If nothing arrives, walk the pipeline backwards: check the routine's last run logs in Claude Code → check `GET /queue` on the Worker → check Fireflies webhook delivery logs.

## Troubleshooting

- **Email lands in spam.** Confirm SPF / DKIM / DMARC for `geekendzone.com` in the Resend dashboard.
- **Worker rejects webhooks (401).** `FIREFLIES_WEBHOOK_SECRET` set in Fireflies must equal the Worker secret. The HMAC header Fireflies sends is `X-Hub-Signature-256`.
- **Routine logs `transcript not ready`.** Fireflies webhook fired before the transcript was queryable. The next tick will retry — no action needed.
- **Trilium 401.** ETAPI token rotated or `TRILIUM_BASE_URL` wrong. Test with `curl -H "Authorization: $TOKEN" $URL/etapi/app-info`.
- **GitHub 422 on PUT.** File already exists with a different sha. The routine handles this; if it persists, check that `GITHUB_TOKEN` has `repo` scope and that the branch is `main`.
